# Base Narrative Rubric

Use this only when a deeper explanation is needed.

## Launchpad Lens

- Clanker: who launched it, who shared it, whether dev publicly recognizes it, whether the meme/product story survives beyond the first day.
- Virtuals: whether the agent has API/demo/revenue/team/ecosystem usage; avoid treating generic AI words as proof.
- Zora: creator/content asset logic; do not evaluate it like an AI agent.
- Flaunch: v4 hook / creator fee / mechanism narrative; check if mechanism creates real demand, not just clever wording.
- Base App: distribution and social entry point matter more than raw meme speed.

## Evidence Strength

Strong:
- CA appears in official X / website / docs.
- Website links to GitHub and GitHub is recently updated.
- X posts include demo, release, docs, product updates, not only price talk.
- Third-party ecosystem/news source discusses the same product narrative.

Medium:
- Website exists and has app/form/API/auth hints.
- GitHub organization exists and recent repos align with the project.
- Token came from recognizable launchpad, but project-specific proof is incomplete.

Weak:
- Same-name GitHub search only.
- Generic launchpad website rather than project token page.
- Thin landing page, no demo, no repo, no X recognition.
- News links are only price pages or mirrors.

## Product Rug Patterns

- story_shell: concept strong, product evidence thin.
- dev_silent: no dev recognition, no updates, no public identity.
- fake_product: landing page exists, but app/backend/GitHub are missing or mismatched.
- repo_mismatch: GitHub found by name only, not explicitly linked.
- launchpad_noise: launchpad metadata exists, but no product-specific evidence.
- liquidity_exit_risk: liquidity is too thin for practical exits.
